# Kiss VPN 0.1.4

Грипчиние gRPC: побочный xray для vless+grpc серверов.

## Что чинит

На некоторых серверах kissmain.ru (например DE-3) вы видели медленные / срывающиеся соединения, в логах xray на стороне сервера:
```
failed to fetch hunk from gRPC tunnel > rpc error: code = Canceled
```

Причина — клиент использовал Mihomo (clash.meta), у которого встроенный gRPC-транспорт работает только в unary-режиме, а серверный xray ждёт bidirectional streaming. Это **архитектурное** ограничение Mihomo, не лечится тюнингом конфига.

## Как починили

Добавили sidecar — официальный **xray-core 26.3.27** — в бандл (~9 MB после сжатия). При подключении клиент сам определяет, какие серверы в подписке используют `network: grpc`, поднимает локальный socks5-инбаунд под каждый такой сервер, проксирует через xray (с `multiMode=true` — bidirectional streaming), а Mihomo ходит к нему как к обычному socks5.

Происходит автоматически, никаких настроек не нужно. Если grpc-серверов в подписке нет — xray не запускается.

Архитектурно: Mihomo всё ещё рулит правилами, TUN, system-proxy. xray — только транспорт для тех серверов, где он реально нужен.

## Технические детали

- При connect: парсим panel Clash YAML → находим `type: vless, network: grpc` → собираем xray JSON → запускаем xray → переписываем grpc-прокси в YAML как socks5-стабы → отдаём Mihomo.
- При disconnect / dispose: xray корректно убивается вслед за Mihomo.
- Probes (тест latency серверов) теперь тоже идут через bridge — grpc-серверы перестанут показывать «∞».

## Обновление

Автообновление подхватит в течение пары часов. Вручную: **Настройки → Обновления → Проверить**.

⚠️ Размер: 51 MB → 60 MB (за счёт встроенного xray).
