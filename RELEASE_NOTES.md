# Kiss VPN 0.1.6

Стабильность стартапа + полноценные логи для багрепортов.

## Что чинит

После багрепортов от юзеров на «резаных» сборках Windows 10 и машинах с параллельно установленными VPN-клиентами (v2RayTun, SnowVPN, Clash for Windows) — клиент включался **2 раза из 10**. Причина — коллизия портов 9090 / 7890 с другим клиентом + молчаливый таймаут без диагностики.

## Что нового

### 🔌 Автоматический выбор свободных портов
- Перед стартом проверяем 9090/7890 (Mihomo controller и mixed-port)
- Если заняты другим VPN-клиентом → автоматически берём 19090/17890, потом 29090/27890 и т.д.
- Системный прокси и WinHTTP правильно указывают на выбранный порт

### ⏱ Раннее обнаружение крэша
- Если Mihomo упал за первые 100 мс — больше не ждём 20 секунд таймаута, сразу выкидываем понятную ошибку
- Увеличили таймаут с 10 до 20 секунд для медленных машин

### 📋 Детальные ошибки
Раньше: `Mihomo did not become ready within 10s`
Теперь: 
```
Mihomo exited prematurely (code 1).
Возможные причины: порт 9090 или 7890 занят другим VPN-клиентом,
отсутствуют geo-файлы, или антивирус блокирует процесс.
Последние строки из ядра:
  panic: listen tcp 127.0.0.1:9090: bind: address already in use
  ...
```

### 📜 Полный лог-сервис для багрепортов
Вкладка **Логи** теперь показывает **два потока**:
- **Клиент** — все события приложения: коннект, выбор портов, ошибки helper, IPC, xray sidecar
- **Ядро** — поток Mihomo (как раньше)

Внизу три кнопки:
- **Очистить** — сбросить буфер
- **В файл** — сохранить лог в `%APPDATA%\com.kissmain\kiss_vpn\logs\`
- **Для багрепорта** — скопировать всё в буфер обмена (включая версию ОС, версию приложения, локаль)

Теперь когда юзер пишет «не работает» — попроси его открыть **Логи → Для багрепорта** и прислать содержимое. Получите полную картину.
